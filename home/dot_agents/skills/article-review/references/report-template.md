# Report Template

Use this as the default output shape. Translate headings to match the user's language if needed, but keep the same section order unless the user asked for a shorter format.

```markdown
# [記事タイトル] 評価レポート

## 判定モード

**[A: 情報・実用重視 / B: 感性・共感重視]**

[必要なら副次モードを 1 行で補足]

## 評価サマリー

- **総合評価**: X.X/5.0
- **炎上リスク**: X.X/5.0（高いほど安全）
- **人間らしさ**: X.X/5.0

### 項目別スコア

- 信頼性と説得力: X.X/5.0
- 構成とストーリーテリング: X.X/5.0
- 読者へのギフト: X.X/5.0
- 表現力と可読性: X.X/5.0
- 温度感と距離感: X.X/5.0
- 炎上リスク・社会的配慮: X.X/5.0
- 人間らしさ: X.X/5.0

## 各項目の詳細評価

### 1. 信頼性と説得力: X.X/5.0

**良い点**

- [具体例]

**改善点**

- [具体例]

**改善案**

- [実行可能な直し方]

[2-7 も同様に続ける]

## 炎上リスク分析

**リスクレベル**: [低 / 中 / 高 / 危険]

**検出されたリスク要因**

- [問題点]
  - 問題の表現: 「[必要最小限の引用]」
  - なぜ問題か: [偏見 / 一般化 / 誤解 / 強すぎる表現など]
  - 改善案: [言い換えや構造修正]

**配慮されている点**

- [具体例]

## 総評

[核となる価値と全体印象を 2-3 文で要約]

## 優先改善提案 TOP3

### 1. [最重要改善点]

**現状**

[何が弱いか]

**改善案**

[何をどう直すか]

**期待効果**

[読者体験や説得力がどう変わるか]

### 2. [次点の改善点]

...

### 3. [3番目の改善点]

...

## リライト例

### Before

> [元の一節]

### After

> [改善後の一節]

**改善ポイント**

- [何を改善したか]
```

## Usage Notes

- For short user requests, compress the prose but keep all major sections.
- If the text has serious verification gaps, add an explicit `未検証の主張` note near the summary or reliability section.
- If the user asked for score-only output, omit the rewrite section and say it was intentionally skipped.
- Quote only the minimum text needed to justify a point.
