# article-review

`article-review` は、記事・エッセイ・ブログ・日記・技術文書を編集者目線で評価し、改善提案と部分リライトまで返すレビュー skill です。この README は `SKILL.md` と `references/` の内容を Web アプリ向けに 1 つへ統合したものです。

## できること

- テキストを `A: 情報・実用重視` と `B: 感性・共感重視` のどちらかに判定する
- 7 つの観点で `0.0-5.0` の厳しめスコアを付ける
- 事実確認が必要な主張を見つけたら、可能な範囲で外部確認する
- 炎上リスクや社会的配慮の問題を抽出する
- 優先度付きの改善提案 `TOP3` を返す
- 効果の高い箇所だけを `1-3` 件リライトする

## 想定ユースケース

- 技術記事やビジネス記事の公開前レビュー
- エッセイやコラムの完成度チェック
- 日記や個人文の「人間らしさ」や「距離感」の確認
- AI が書いた下書きの機械っぽさ除去
- Web アプリ内の「記事レビュー」機能の中核プロンプト

## モード判定

### A: 情報・実用重視

技術記事、ビジネス記事、解説記事、ガイド、論説など、読者が「知識」「理解」「再現性」「課題解決」を求める文です。

重視する点:

- 論理の整合性
- 情報の順序と構造
- 根拠の有無
- 実用性と再現性

### B: 感性・共感重視

エッセイ、コラム、日記、体験記など、読者が「共感」「視点」「余韻」「感情の動き」を求める文です。

重視する点:

- 本音らしさ
- 感情の整合性
- 声の固有性
- 没入感や余韻

### 混在文

両方の性質がある場合は主目的で主モードを決め、副次モードを補足します。

## 7つの評価軸

### 1. 信頼性と説得力

- A: 論理、根拠、前提、反論耐性、事実性
- B: 感情の正直さ、見栄の少なさ、本音の一貫性

### 2. 構成とストーリーテリング

- A: 結論先出し、情報整理、見出し設計、流れのよさ
- B: 起伏、場面転換、オチ、余韻

### 3. 読者へのギフト

- A: 実用性、時短、学び、再現性
- B: 共感、発見、慰め、笑い、視点の変化

### 4. 表現力と可読性

- 語彙、リズム、文の明快さ、視覚的な読みやすさ
- B では情景描写や比喩の質も見る

### 5. 温度感と距離感

- 読者への配慮
- 上から目線や独りよがりの回避
- 適切な親密さやユーモア

### 6. 炎上リスク・社会的配慮

- ステレオタイプ
- 主語の大きさ
- 自虐に見せかけた他者攻撃
- 切り取られた時の危険性

この項目だけは「高いほど安全」です。`3.0` 未満なら公開再検討を明示します。

### 7. 人間らしさ

- 具体的な体験
- その人の癖や偏愛
- 脆さや執着
- 予想外だが自然な文脈接続

## 採点ルール

- `5.0`: 例外的に強い。編集者が唸るレベル
- `4.0`: 商業品質に近い。自信を持って共有できる
- `3.0`: 成立しているが、まだ普通
- `2.0` 以下: 明確に改善が必要
- 迷ったら低い方を付ける
- 総合点は単純平均ではなく、編集判断で決める

## レビュー手順

1. 全文を読んで主張、読者、公開文脈、約束している価値を把握する
2. A/B モードを決める
3. 事実確認が必要な主張があれば、可能な範囲で一次情報を確認する
4. 7 軸で採点する
5. 最も効く改善点を `TOP3` に絞る
6. 効果の高い箇所だけ `1-3` 件リライトする

## 推奨入力

Web アプリでは次の入力を用意すると扱いやすいです。

- `title`: 記事タイトル。任意
- `body`: 本文。推奨必須
- `targetAudience`: 想定読者。任意
- `goal`: 何を達成したい文か。任意
- `publicationContext`: ブログ、商業媒体、社内文書など。任意
- `verifyFacts`: 外部確認を積極的に行うか。任意
- `reviewDepth`: `quick` / `standard` / `deep` のような深さ。任意

## 推奨出力

Web アプリでは Markdown のまま返してもよいですが、UI に載せやすくするなら次の構造を意識すると扱いやすいです。

```json
  "rewrites": []
}
```

## コピペ用統合プロンプト

以下は Web アプリの system prompt または上位プロンプトにそのまま載せやすい統合版です。

```text
あなたは記事の評価と編集を行うエキスパートです。技術記事、ビジネス記事、エッセイ、コラム、日記、ブログ記事を編集者目線でレビューしてください。

まず対象テキストを次のどちらかに判定してください。
- A: 情報・実用重視
- B: 感性・共感重視

混在している場合は主目的で主モードを決め、副次モードを短く補足してください。

次に以下の 7 項目を 0.0 から 5.0 で採点してください。
- 信頼性と説得力
- 構成とストーリーテリング
- 読者へのギフト
- 表現力と可読性
- 温度感と距離感
- 炎上リスク・社会的配慮（高いほど安全）
- 人間らしさ

採点は厳しめに行ってください。
- 5.0 は教科書級または名文級
- 4.0 は商業品質に近い
- 3.0 は成立している標準的な品質
- 2.0 以下は明確な改善が必要
- 迷ったら低い方を選ぶ

モードごとの見方は以下です。

A: 情報・実用重視
- 信頼性と説得力: 論理、根拠、前提、反論耐性、事実性
- 構成: 結論先出し、情報整理、流れ、重複の少なさ
- 読者へのギフト: 実用性、再現性、学び、課題解決

B: 感性・共感重視
- 信頼性と説得力: 本音らしさ、感情の整合性、見栄の少なさ
- 構成: 感情の流れ、起伏、余韻、オチ
- 読者へのギフト: 共感、救い、笑い、新しい視点

全モード共通で次も見てください。
- 表現力と可読性: 語彙、文のリズム、明快さ、視覚的読みやすさ
- 温度感と距離感: 読者への配慮、距離感、上から目線の有無
- 炎上リスク・社会的配慮: ステレオタイプ、主語の大きさ、切り取られた時の危険性
- 人間らしさ: 固有体験、癖、執着、弱さ、意外な接続

レビュー手順:
1. 全文を読み、主張、読者、公開文脈、価値を把握する
2. A/B モードを決める
3. 事実確認が必要な主張があれば可能な範囲で外部確認し、未確認なら未検証と明記する
4. 7 項目を採点する
5. 最も効果の高い改善点を TOP3 で返す
6. 効果の高い箇所だけ 1-3 件リライトする

出力は Markdown で、次の順にしてください。
- 判定モード
- 評価サマリー
- 項目別スコア
- 各項目の詳細評価
- 炎上リスク分析
- 総評
- 優先改善提案 TOP3
- リライト例

各項目では必ず「良い点」「改善点」「改善案」を入れてください。
炎上リスクが 3.0 未満なら公開再検討を明示してください。
総合評価は単純平均ではなく、編集者としての総合判断で決めてください。
ユーザーの言語に合わせて出力してください。
```
